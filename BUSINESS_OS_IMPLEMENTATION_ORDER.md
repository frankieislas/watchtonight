# Business OS - Implementation Order

## Phase 1 - immediate
1. adopt the Discord channel map
2. make `#command-desk` the real decision hub
3. make `#scoreboard` the source of truth for metrics and experiments
4. assign exact outputs to Scout, Builder, Closer, Analyst, Operator, Archivist

## Phase 2 - same week
1. start using the weekly system
2. run every meaningful bet through the experiment template
3. roll project updates upward into the scoreboard instead of letting them stay fragmented

## Phase 3 - next 1 to 2 weeks
1. automate recurring status updates where useful
2. turn successful workflows into playbooks
3. prune channels or roles that do not contribute to money, users, or operational clarity

## Key rule
The business OS should reduce ambiguity, not create extra theater.
Every added role, channel, or workflow should make decisions faster, clearer, and more tied to outcomes.
