# Business OS - Discord Map

## Purpose
This Discord structure is designed to operate like a business command system, not just a chat server.

## Core categories
### 1. Command Desk
- `#command-desk` - top-level priorities, approvals, decisions, active focus
- `#weekly-review` - weekly recap, lessons, keep/kill/pivot decisions
- `#scoreboard` - current business metrics, experiments, top bets

### 2. Execution Lanes
- `#scout` - opportunities, market ideas, leads, demand signals
- `#builder` - product shipping, bugs, technical progress, implementation
- `#closer` - offers, pricing, conversion, sales ideas, monetization tests
- `#analyst` - metrics, experiment results, dashboards, readouts
- `#operator` - systems, automations, infrastructure, workflows
- `#archivist` - docs, playbooks, lessons, memory, system updates

### 3. Project Rooms
Create one channel per serious project:
- `#watchtonight`
- `#project-name`

Use these only for project-specific execution, not for top-level business decisions.

### 4. Distribution / Growth
- `#distribution` - outreach, content, partnerships, acquisition experiments
- `#user-feedback` - user quotes, objections, reactions, qualitative insight
- `#offers` - premium concepts, landing hooks, signup and pricing tests

### 5. Ops / Logs
- `#ship-log` - notable launches and shipped changes
- `#wins` - progress worth retaining morale around
- `#blockers` - unresolved blockers needing decision or intervention

## Rule of use
- strategic decisions happen in `#command-desk`
- metrics roll up into `#scoreboard`
- each role updates its own lane
- projects do not become the operating system, they plug into it
