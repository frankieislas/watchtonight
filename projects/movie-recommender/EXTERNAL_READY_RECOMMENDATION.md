# External-Ready Recommendation

## Goal
Choose the smallest practical external-ready implementation path for WatchTonight without publishing yet.

## Recommended hosting option
### GitHub Pages

Why this is the current recommendation:
- simple static hosting
- fast to use for a landing page
- cheap and lightweight
- fits the current HTML-first waitlist page
- enough for validation without app complexity

## Recommended email-only form backend option
### Formspree-style lightweight form backend

Why this is the current recommendation:
- simplest realistic way to collect email signups from a static page
- avoids building a custom backend
- enough for waitlist validation
- low operational overhead

## Recommended pairing
- Hosting: GitHub Pages
- Form backend: lightweight form service for email-only signup

## Why this pair
- smallest practical path to a real public waitlist page
- minimal infrastructure
- easy to maintain
- aligned with the current static HTML page
- fast to test once approved

## Boundary
This is a preparation recommendation only.
No external account creation, configuration, or publishing should happen without explicit user approval.
