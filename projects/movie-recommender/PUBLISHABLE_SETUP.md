# Publishable Setup Plan

## Goal
Prepare WatchTonight for the simplest realistic launch path without publishing yet.

## Recommended approach
Use:
- a static landing page
- email-only signup
- the lightest possible form backend
- simple hosting when approved

## Why this path
- fastest to launch
- cheapest to maintain
- enough for validation
- avoids overbuilding too early

## Minimal launch stack
### Frontend
- static HTML landing page

### Signup collection
- email-only form submission
- lightweight form backend or equivalent simple collection path

### Hosting
- simple static hosting
- no custom app backend yet

## What needs to exist before public launch
- final approved page copy
- working signup submission path
- success state after signup
- clear place where collected emails will be accessible

## What does not need to exist yet
- user accounts
- login
- app dashboard
- recommendation engine integration
- billing

## Current state
- landing page draft exists
- email-only strategy is chosen
- signup collection path is planned
- publishable path is not yet wired

## Next implementation layer
1. choose a specific simple publish path
2. choose a specific form collection method
3. wire form action/success handling
4. test privately
5. ask for final approval before publishing
