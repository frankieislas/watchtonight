# Single Scoreboard Spec

## Purpose
One page that tells Atom what matters right now across products and experiments.

## Top-level sections
### 1. Active projects
For each project:
- stage
- current goal
- owner
- next milestone
- blocker

### 2. Growth / user signals
- testers contacted
- testers active
- repeat users
- signups
- conversion signals
- strongest user segment

### 3. Revenue signals
- willingness to pay count
- offers tested
- conversions
- revenue so far
- next revenue test

### 4. Experiment tracker
- experiment name
- hypothesis
- start date
- current status
- result
- next action

### 5. Risk / blockers
- technical blockers
- distribution blockers
- monetization blockers
- decisions needed

## Best implementation
Start simple.
Use a markdown file or lightweight table first before building a dashboard product.
