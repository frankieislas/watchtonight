# MVP Build Spec

## Product goal
Help users decide what movie to watch tonight quickly and confidently.

## MVP outcome
A user can choose their streaming services, share basic taste preferences, and receive 3 personalized movie picks with reasons and watch availability.

## MVP scope

### User inputs
- streaming services available
- favorite genres
- disliked genres
- optional mood or vibe
- optional watched/rated feedback

### Core outputs
- top 3 movie recommendations
- where to watch
- short reason each movie fits
- one best overall pick

### Personalization layer
- store basic preference profile
- use feedback from watched/rated movies
- avoid repeated dislikes when possible

### Recommendation inputs
- streaming availability
- trend/discussion signals
- quality/reputation signals
- learned user taste

## Non-goals for MVP
- TV show recommendations
- social feed
- advanced account system
- mobile app
- friend sharing
- large-scale discovery browsing

## Suggested MVP surfaces
Priority order:
1. landing page + waitlist
2. simple web app or concierge experience
3. lightweight account/profile memory

## Key UX promise
The product should help a user choose a movie in under a minute.

## Open questions for later
- best source for streaming availability
- best source for trend signals
- whether the first product should be web-first or messaging-first
- when to add household/couple profiles
