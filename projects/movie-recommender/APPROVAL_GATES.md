# Approval Gates

## Principle
No public release, no external account creation, and no spend without explicit user approval.

## Current approved actions
- internal planning
- local file creation
- workspace edits
- local git commits

## Future actions that require approval
- creating or using external hosting
- creating or using a form backend / external collection tool
- publishing the waitlist page publicly
- sharing the page externally
- spending money on infrastructure, domains, or tools

## Next approval gate
Before any public launch, the user must approve:
1. the chosen publish path
2. the chosen signup collection method
3. the go-live step
