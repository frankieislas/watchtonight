# Waitlist Flow

## Goal
Capture interest before building the full product and learn which positioning resonates most.

## Core conversion goal
Get users to join an early access list for WatchTonight.

## Recommended flow

### 1. Landing page
User lands on a simple page with:
- headline
- subheadline
- short explanation of how it works
- streaming-service support line
- early access CTA

### 2. Signup form
Collect the minimum information needed:
- email address
- optional first name
- optional question: Which streaming services do you use most?
- optional question: What is hardest about picking a movie?

### 3. Confirmation page
After signup, show:
- confirmation message
- short explanation of what happens next
- optional prompt to share with a friend

## Suggested form fields
Required:
- Email

Optional:
- First name
- Streaming services used most
- Main frustration when choosing a movie

## Confirmation message draft
You're in.

WatchTonight is being built to help you skip the endless scrolling and get 3 strong movie picks you can actually watch tonight.

We’ll let you know when early access opens.

## What to learn from waitlist responses
- Which streaming services are most common
- Whether people care more about speed, personalization, or quality filtering
- What language people use to describe the problem
- Whether this feels like a real pain or just a nice-to-have

## Success signals
- signups from cold visitors
- people describing the same pain clearly
- users caring about personalization + subscription filtering
- users asking when they can try it
