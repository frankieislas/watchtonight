# WatchTonight MVP Build Plan

## Goal
Build the first real usable version of WatchTonight after initial waitlist validation.

## Product goal
Help a user decide what movie to watch tonight in under a minute.

## Recommended first build surface
- simple web app

## Why web first
- fastest to build
- easiest to test with waitlist users
- enough to prove product value
- avoids mobile-app overhead too early

## Core MVP user flow
1. user lands on the product
2. user selects the streaming services they have
3. user shares basic taste preferences
4. user gets top 3 movie recommendations
5. user sees where to watch and why each fits
6. user can leave feedback on what they watched / liked / disliked

## MVP must-have features
- streaming service selector
- basic taste preference input
- recommendation result view
- top 3 movie picks
- where to watch each pick
- short fit explanation for each pick
- one best overall pick
- lightweight taste memory / profile update flow

## MVP should avoid for now
- TV shows
- heavy social features
- broad catalog browsing
- complex account architecture
- native mobile app
- complex billing system

## Build priority
### Phase 1
- recommendation input form
- recommendation output screen
- basic hosted app shell

### Phase 2
- lightweight taste profile memory
- feedback submission flow
- better recommendation refinement

### Phase 3
- tighter onboarding
- saved user state if needed
- beta polish

## Validation question after MVP
Does a user actually come away with a faster, better decision than they get from browsing streaming apps alone?
