# Signup Collection Plan

## Goal
Choose the simplest way to capture early interest for WatchTonight without overbuilding.

## Recommendation
Start with a very simple signup flow:
- required field: email
- optional fields later if needed
- no account system yet
- no complex onboarding yet

## Best first version
### Form fields
Required:
- Email address

Optional later:
- First name
- Which streaming services do you use most?
- What is hardest about choosing a movie?

## Why start simple
- lowest friction
- easier to test interest
- faster to launch
- cleaner validation signal

## Recommended sequence
1. launch with email-only capture
2. see if people actually sign up
3. add optional questions only if needed for learning

## Publishing paths to consider later
- simple static page with form backend
- GitHub Pages + lightweight form service
- other static hosting + form handler

## What not to do yet
- full auth
- user dashboards
- custom backend for accounts
- complex onboarding funnel

## Next approval questions
1. Do we want email-only signup first?
2. Do we want a real form backend next, or just keep the draft local until the page is stronger?
3. When we publish, do we want the simplest static-hosted path possible?
