# Business OS - Production Discord Structure

## Category 1 - Command
### `#command-desk`
Use for:
- top priorities
- approvals
- decisions
- high-level coordination

### `#scoreboard`
Use for:
- current numbers
- active experiments
- top bets
- weekly movement

### `#weekly-review`
Use for:
- weekly summary
- wins, misses, lessons
- keep/kill/pivot decisions

## Category 2 - Execution Roles
### `#scout`
Use for:
- opportunities
- new ideas
- market signals
- target user segments

### `#builder`
Use for:
- product execution
- bugs
- shipping updates
- implementation blockers

### `#closer`
Use for:
- pricing
- offers
- monetization tests
- conversion ideas

### `#analyst`
Use for:
- metrics
- readouts
- experiment summaries
- scoreboard support

### `#operator`
Use for:
- systems
- automations
- recurring workflows
- reliability

### `#archivist`
Use for:
- playbooks
- memory
- docs
- lessons captured

## Category 3 - Projects
One channel per meaningful project.
Examples:
- `#watchtonight`
- `#project-x`

Rule:
Projects are execution containers, not decision centers.
Major decisions roll up to `#command-desk`.

## Category 4 - Growth / Users
### `#distribution`
- outreach
- content tests
- audience growth
- partnerships

### `#user-feedback`
- quotes
- objections
- feature requests
- signup signals

### `#offers`
- premium language
- pricing tests
- landing hooks

## Category 5 - Signal / Logs
### `#ship-log`
- meaningful launches
- changes shipped
- key product updates

### `#wins`
- useful wins worth reinforcing

### `#blockers`
- unresolved problems needing decisions or intervention
