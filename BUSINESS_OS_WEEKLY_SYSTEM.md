# Business OS - Weekly System

## Monday
- choose top bets
- assign owners
- define success criteria

## Tuesday to Thursday
- execute
- ship
- recruit users
- run experiments
- log blockers fast

## Friday
- update scoreboard
- review growth, usage, and money signals
- decide what is real

## Saturday
- improve the system
- document lessons
- clean up workflows and notes

## Sunday
- make keep/kill/pivot decisions
- define the next cycle

## Weekly command desk questions
- what moved revenue odds?
- what moved user demand?
- what was noise?
- what should stop immediately?
- where should more focus go next?
