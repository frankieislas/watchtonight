# Business OS - Role Workflow Map

## Operating principle
Each role exists to produce outputs, not just conversation.

## Scout
### Inputs
- market noise
- trends
- competitor movement
- user pain points

### Outputs
- opportunity briefs
- user segment hypotheses
- demand signals
- lead lists

## Builder
### Inputs
- approved experiments
- product priorities
- blockers

### Outputs
- shipped features
- technical fixes
- instrumentation
- implementation status

## Closer
### Inputs
- product capability
- user pain points
- value signals

### Outputs
- offer tests
- pricing tests
- premium positioning
- conversion language

## Analyst
### Inputs
- product usage
- tester feedback
- signup data
- conversion data

### Outputs
- scoreboards
- experiment readouts
- recommendations on what to stop or scale

## Operator
### Inputs
- workflow friction
- repeated tasks
- coordination gaps

### Outputs
- automations
- process cleanup
- systems reliability
- recurring review flows

## Archivist
### Inputs
- shipped work
- lessons learned
- experiment results
- decisions made

### Outputs
- memory updates
- playbooks
- decision history
- clean documentation

## Handoff model
- Scout identifies opportunities
- Command desk chooses bets
- Builder executes
- Analyst evaluates
- Closer monetizes
- Operator improves system
- Archivist preserves what matters
